Classifying plant seedlings using Fastai

Oct 10, 2020 â€¢ 6 min read

This notebook demonstrates Kaggle notebook + Fastai version 2 functionality for Plant Seedlings competition (https://www.kaggle.com/c/plant-seedlings-classification) from scratch

Import fastai vision library 

from fastai.vision.all import *

Add the data using Add data button to the right of the notebook. You should select Competition data tab and search for plant seedlings. Set the parent directory to Path object. This way, we get to access the directory in a much efficient way. For example, path.ls() would provide subfolders inside the parent directory** 

path = Path('/kaggle/input/plant-seedlings-classification')
path.ls()

Instantiate a DataBlock object and choose the arguments as per the problem in hand:

  • blocks: since this a multi-class classification problem, we require Imageblock and CategoryBlock

  • get_items: to feth all images in the parent folder

  • 'splitter': in the original dataset, there is no validation sub-folder and thus, we would use this argument to split training data to form a valid set.

  • get_y: images are arranged in folders named as labels for the images

  • item_tfms: images are of varying sizes and we use this argument to resize them to 300 x 300

  • batch_tfms: this augmentation applied to a mini-batch. Normalize would subtract mean from each pixel value and divide by the standard deviation (all computations are done internally) and size is mainly for progressive resizing
def get_dls(bs, size):
    block = DataBlock(blocks=(ImageBlock, CategoryBlock),
                 get_items=get_image_files,
                 splitter=RandomSplitter(valid_pct=0.2, seed=40),
                 get_y=parent_label,
                 item_tfms=Resize(300),
                     batch_tfms=[*aug_transforms(size=size, min_scale=0.75),
                               Normalize])
    return block.dataloaders(path/"train", bs=bs)

Let's see what the Dataloaders object has by plotting a batch using show_batch. Even though the batch has 64 images, Fastai shows a 3 x 3 subplot to give any idea of how the images look like 

dls = get_dls(64, 128)
dls.show_batch()

As the next step, let's define a Learner: I chose model = resnet50, loss = CrossEntropyLossFlat. Before training, I also ran lr_find that is a great feature of Fastai which recommends the learning rates for a training

  • Note: For the model to be downloaded, you must turn ON internet from Settings/Internet available in the right pane.
learner1 = cnn_learner(dls, resnet50, CrossEntropyLossFlat(), metrics=error_rate)
learner1.lr_find()

To this end, we tried 3 approaches to this problem:



1. Progressive resizing
2. Label smoothing
3. Mixup



Below are some of the key observations:


    

  • It takes around the same time per epoch for each of the approaches
    • 

  • The least error_rate of 2.3% was achieved with Label smoothing when Resnet50 was trained for 30 epochs
    • 

  • Comparing losses for label smoothing and Mixup, we can see that both these approaches avoid overfitting much effectively as compared to progressive resizing
    • 

  • Progressive sizing may not be an ideal solution because overfitting can be observed with every resizing
    •
